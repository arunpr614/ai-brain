Android Library Filter UX Options

The current filter block solves discoverability but takes over the first mobile viewport. These throwaway prototypes compare three directions where filters are available by default, then can get out of the user's way.

My recommendation

Use Option B as the final pattern: keep search and current filter state visible, move filter choice into an explicit bottom sheet. It uses the least vertical space and feels native to Android.

Option A

Expandable Filter Tray

Shows one compact status row by default. The user can expand filters, choose a value, then collapse the tray.
