## §3. Bucket A failure paths

| If A-1 fails | Likely cause |
|---|---|
| Brain opens to library, no /items/<id> route | share-handler not hydrated; SW + chunks still loading; retry |
| Server logs no /api/capture POST | bearer token missing; re-pair via /setup-apk |

| If A-2 fails | Likely cause |
|---|---|
| ERR_INTERNET_DISCONNECTED | SW didn't register OR cache.addAll never completed; check chrome://inspect Application tab |
| Brain opens but no toast + no /inbox row | share-handler hydrated but outbox path didn't fire; check console for IDB errors |
| Toast appears but /inbox shows empty | initOutbox() race — share-handler fell through to direct-POST path; check `share.outbox.init-failed` in logcat |

| If A-3 fails | Likely cause |
|---|---|
| Row stays queued after airplane off | @capacitor/network listener didn't register OR stuck-state notification blocking sync; check `triggers.ts` events |
| Row goes stuck:auth_bad | bearer token expired during offline window; re-pair |
| Row goes stuck:version_mismatch | EXPECTED_CLIENT_API mismatch (shouldn't happen — Lane C hasn't bumped) |
