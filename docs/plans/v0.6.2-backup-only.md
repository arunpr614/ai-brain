## T-1 — D-18 B2 off-site backup

- **File:** `src/lib/backup.ts` — add a post-`VACUUM INTO` step in `runBackupOnce()`. Encrypt the just-written snapshot with `gpg --encrypt --recipient $BACKUP_GPG_RECIPIENT --output <dest>.gpg <dest>`, upload `<dest>.gpg` to B2 bucket `brain-backups` via the official `@backblaze-b2` Node SDK, then `rm <dest>.gpg`. Cleartext local snapshot retained for the 28-snapshot rotation. No B2 code path exists today — this is net-new.
- **Config:** read `B2_KEY_ID`, `B2_APP_KEY`, `B2_BUCKET`, `BACKUP_GPG_RECIPIENT` from `/etc/brain/.env`. If any are missing, log `[backup] off-site disabled — <missing var>` and continue local-only (don't crash the scheduler).
- **Failure semantics:** B2 upload errors logged as `[backup] off-site upload failed: <msg>` and swallowed — local snapshot still counts as a backup; off-site is best-effort. No retry in v0.6.2 (add only if real failures observed).
- **GPG escrow:** 1Password (matches D-1..D-4 escrow pattern). Export armored private key + revocation cert into the existing `Brain — secrets` vault item before T-1 ships.
- **Test:** `src/lib/backup.test.ts` — stub the B2 SDK + `gpg` invocation; assert (a) upload called with the encrypted artifact, (b) cleartext local snapshot still on disk, (c) missing-env path logs warning + skips upload without throwing, (d) B2 upload throw is caught and logged.
- **Acceptance (D-18 smoke):** SSH Hetzner → trigger `runBackupOnce()` (via existing 6h cron or one-shot) → `b2 ls brain-backups` shows `<timestamp>.sqlite.gpg` → `b2 download-file-by-name brain-backups <name> /tmp/restore.gpg` → `gpg --decrypt /tmp/restore.gpg > /tmp/restore.sqlite` → `sqlite3 /tmp/restore.sqlite 'select count(*) from items'` matches live DB (9). UI: `/settings` line at `src/app/settings/page.tsx:102` updated — drop "Off-site backup pending (see roadmap v0.6.2)".
