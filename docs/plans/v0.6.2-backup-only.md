## T-G — GPG escrow gate (must pass before any off-site backup is meaningful)

Backups encrypted with a key that exists only on this Mac are unrecoverable on Mac loss. Block T-1 acceptance on:

1. `gpg --export-secret-keys --armor BC1CCA584E82D84B > /tmp/private.asc`
2. `gpg --gen-revoke BC1CCA584E82D84B > /tmp/revoke.asc`
3. Paste both into 1Password `Brain — secrets` (matches D-1..D-4 escrow pattern).
4. `rm -P /tmp/private.asc /tmp/revoke.asc` (secure-delete).
5. Confirm with user that the paste landed.

**Passphrase status:** key is passphrase-protected (passphrase stored in 1Password alongside the key). Two-wall escrow: 1Password account access (wall 1) + passphrase string (wall 2). Encryption on Hetzner uses the **public** key — cron job needs no passphrase plumbing. Passphrase is only required at restore time on Mac.

## T-1 — D-18 rclone+cron on Hetzner

Files (all server-side; nothing in this repo's `src/`):

- **`scripts/backup-offsite.sh`** *(new in repo, deployed to `/opt/brain/scripts/backup-offsite.sh`)*
  - `set -euo pipefail`
  - `SNAPSHOT=/opt/brain/data/backups/$(date -u +%Y-%m-%d_%H%M).sqlite`
  - `sqlite3 /opt/brain/data/brain.sqlite ".backup '$SNAPSHOT'"` (WAL-safe; matches in-process scheduler)
  - `gpg --batch --yes --trust-model always --encrypt --recipient BC1CCA584E82D84B --output "$SNAPSHOT.gpg" "$SNAPSHOT"`
  - `rclone copyto "$SNAPSHOT.gpg" "b2:$B2_BUCKET_NAME/$(basename "$SNAPSHOT.gpg")"` (reads existing rclone B2 remote)
  - `rm -f "$SNAPSHOT.gpg"` (cleartext snapshot stays — feeds the existing 28-snapshot in-process rotation)
  - All output appended to `/var/log/brain-backup.log` (logrotate'd by host default).
  - Failure modes: missing rclone → exit 1 + log; gpg fail → exit 1 + log; B2 upload fail → exit 1 + log. **Cleartext local snapshot still exists** (best-effort off-site, on-disk is the authoritative copy).

- **`/etc/cron.d/brain-backup`** *(host-only, never in repo)*
  - `0 */6 * * * brain /opt/brain/scripts/backup-offsite.sh >> /var/log/brain-backup.log 2>&1`
  - Cadence matches existing in-process 6h scheduler. Both run; in-process is the local-rotation engine, cron is the off-site engine. Drift between the two is harmless (each call to `.backup` is independent).

- **Hetzner provisioning (one-shot, manual)**

  - `sudo -u brain rclone config` — add B2 remote named `b2` using existing `B2_KEY_ID` / `B2_APPLICATION_KEY` / `B2_ENDPOINT` / `B2_BUCKET_NAME` from `/etc/brain/.env`.
  - B2 lifecycle rule: 30-day expiry on the bucket (one-time, via B2 dashboard — no code).

  - Note: backup is server-side; local dev does not need these set.
  - Drop the inherited "Off-site backup pending" copy from `src/app/settings/page.tsx:101-103` (already removed in the reverted commit; needs re-applying).

- **No new npm deps.** Pure shell + system rclone + system gpg. Fixes the 5-warning audit hit from the SDK path.

**Acceptance (D-18 smoke, full restore round-trip):**
1. SSH Hetzner → `sudo -u brain /opt/brain/scripts/backup-offsite.sh` (manual one-shot, not waiting for cron).
