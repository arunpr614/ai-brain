## 1. Rationale and pivot summary

The v1.3 LAN-only plan used mDNS (`brain.local` via `bonjour-service`) to make the Mac's Next.js server discoverable over home Wi-Fi. At the T-21 AVD smoke gate, the macOS Application Firewall blocked inbound connections from the Android Pixel; configuring an allow-rule for `node` required navigating macOS System Settings and was outside the user's comfort zone.

**Pivot chosen:** Cloudflare named tunnel (`brain` / UUID `58339d22-d0be-4fab-94d6-32fd24b04a72`) at stable domain `https://brain.arunp.in`.

**Why named tunnel over quick tunnel:** quick tunnels do NOT support Server-Sent Events (confirmed from official Cloudflare docs: "Quick Tunnels do not support Server-Sent Events (SSE)"). The v0.4.0 Ask feature depends on SSE (`/api/ask`). Named tunnels support SSE — empirically verified 2026-05-11 via `curl -N https://brain.arunp.in/api/ask` (token-by-token stream, `ttfb == total`, no buffering). See RUNNING_LOG.md entry 28.

**Why cloudflared outbound-only matters:** `cloudflared` makes exclusively outbound QUIC connections to Cloudflare edge (port 7844, destinations `region1.v2.argotunnel.com` and `region2.v2.argotunnel.com`). The macOS Application Firewall only intercepts inbound connections, so no firewall prompt appears — the entire pivot motivation is resolved.

### T-CF-4 — QR schema change: `url=` replaces `ip=`

**Scope:** Update `src/lib/lan/setup-uri.ts` — the pure QR URI parser — to accept the new schema:

```
brain://setup?url=https://brain.arunp.in&token=<64-hex-lowercase>
```

Rules:
- `url` field: must start with `https://`; hostname must be non-empty; reject `http://`. Full URL stored; parsed via `new URL()`.
- `token` field: 64-char hex lowercase (unchanged from LAN-era).
- `ip` field: no longer accepted. If present in an old QR scan, emit `verdict.kind === 'invalid'` with reason `"ip-field-deprecated"`.
- `buildSetupUri(token)` signature changes from `(ip, token)` to `(token)` — `url` is always the constant `BRAIN_TUNNEL_URL`.

**Files:** `src/lib/lan/setup-uri.ts` (EDIT), `src/lib/lan/setup-uri.test.ts` (REWRITE — 12 LAN-era tests replaced by 12 tunnel-era tests).

**Acceptance criteria:**
1. `npm test` green — all 12 new tests pass, no regressions.
2. `buildSetupUri("<token>")` returns `"brain://setup?url=https://brain.arunp.in&token=<token>"`.
3. `parseSetupUri("brain://setup?url=https://brain.arunp.in&token=<valid>")` returns `{kind: 'ok', url: 'https://brain.arunp.in', token: '<valid>'}`.
4. `parseSetupUri("brain://setup?ip=192.168.1.17&token=<valid>")` returns `{kind: 'invalid', reason: 'ip-field-deprecated'}`.
5. `parseSetupUri("brain://setup?url=http://brain.arunp.in&token=<valid>")` returns `{kind: 'invalid', reason: 'url-not-https'}`.

## 7. SSE smoke script spec

Per SPIKE-003: the existing unit test suite cannot verify streaming behavior (tests use `res.text()` which buffers the whole response). A manual smoke script is the required gate.

### `scripts/smoke-sse.sh`

```bash
#!/usr/bin/env bash
# scripts/smoke-sse.sh — SSE streaming verification via real Ollama inference
# Usage: BRAIN_URL=https://brain.arunp.in BRAIN_TOKEN=<token> bash scripts/smoke-sse.sh
#
# Expected output: lines with progressive timestamps (e.g., [10:14:32.1], [10:14:32.4], ...)
# FAIL signal: all lines share the same timestamp — response was buffered, not streamed.
#
# Request body note: /api/ask expects {"question": "...", "thread_id": "<string>"}
# NOT {"query": "..."} — see RUNNING_LOG entry 28 "Learned" (bearer check found this).
# thread_id must be a string (e.g., "smoke-test-001"), not null.
#
# The question should be long enough to produce >90s Ollama inference if possible,
# to exercise the originRequest.keepAliveTimeout: 10m tunnel config.
# A ~500-word essay prompt achieves this on qwen2.5:7b-instruct-q4_K_M.

set -euo pipefail
: "${BRAIN_URL:?required — e.g. https://brain.arunp.in}"
: "${BRAIN_TOKEN:?required — 64-char hex from .env or rotate-token.sh}"

QUESTION="${1:-Write a 500-word essay on the history of the internet, covering ARPANET, TCP/IP, the World Wide Web, and the smartphone era. Be thorough.}"

echo "Smoke SSE: ${BRAIN_URL}/api/ask"
echo "Question: ${QUESTION:0:80}..."
echo "Watch timestamps — they should span multiple seconds for a 500-word essay."
echo "If all timestamps are identical, the response is buffered (FAIL)."
echo "---"

curl -s -N "${BRAIN_URL}/api/ask" \
  -H "Content-Type: application/json" \
  -H "Cookie: brain-session=<obtain-cookie-from-browser-session>" \
  -d "{\"question\": \"${QUESTION}\", \"thread_id\": \"smoke-test-$$\"}" \
  | while IFS= read -r line; do
      printf "[%s] %s\n" "$(date +%H:%M:%S.%N | cut -c1-12)" "$line"
    done
```

**Note on authentication:** `/api/ask` is cookie-only (CF-D-08). The smoke script requires a `brain-session` cookie from a browser-authenticated session. To obtain:
1. Open `https://brain.arunp.in` in a browser; enter PIN.
2. Inspect DevTools → Application → Cookies → copy `brain-session` value.
3. Substitute `<obtain-cookie-from-browser-session>` in the script OR use the `-b cookie.txt` flag if you exported the cookie.

**>90s keepalive test:** The `originRequest.keepAliveTimeout: 10m` config in `~/.cloudflared/config.yml` is intended to keep SSE connections alive for long inferences. To verify empirically: the 500-word essay prompt on `qwen2.5:7b-instruct-q4_K_M` takes approximately 2-5 minutes on an M1 Pro. If the stream completes without a `curl: (18) transfer closed with outstanding read data` error, the keepalive setting is working.

**Wired as:** `"smoke:sse": "BRAIN_URL=https://brain.arunp.in BRAIN_TOKEN=${BRAIN_LAN_TOKEN:-} bash scripts/smoke-sse.sh"` in `package.json`.

## 10. Threat model update

The pivot from LAN-only to public HTTPS changes the attack surface. The bearer token is no longer protected by LAN proximity — it is now a credential that, if leaked, is directly exploitable over the public internet.

### Trust boundaries (post-pivot)

| Boundary | Description |
|---|---|
| `brain.arunp.in` (Cloudflare edge) → `localhost:3000` | Cloudflare terminates TLS; request forwarded via outbound QUIC tunnel. Cloudflare can observe plaintext. |
| Internet → `brain.arunp.in` | Public HTTPS; any IP can reach the API. Auth gate is the only control. |
| APK WebView → `brain.arunp.in` | Cookie-authed for browser navigation; bearer-authed for capture routes. Cookie is SameSite=Strict. |
| Chrome extension → `brain.arunp.in` | Bearer-authed only. Token stored in `chrome.storage.local` (local only, not synced). |

### STRIDE threat register

| Threat ID | Category | Component | Disposition | Mitigation |
|---|---|---|---|---|
| T-CF-S-01 | Spoofing | Bearer token leaked via extension storage | mitigate | `chrome.storage.local` is profile-local; not synced; not accessible to web pages. Token rotation via `scripts/rotate-token.sh` or `/api/settings/rotate-token`. |
| T-CF-S-02 | Spoofing | Bearer token brute force over public internet | accept | 256-bit entropy token; at any req/s rate would take astronomically long. Not a realistic threat. |
| T-CF-T-01 | Tampering | Cloudflare edge modifies request/response | accept | Cloudflare is the chosen transport; user accepted this trust model. Personal notes app; risk commensurate. |
| T-CF-T-02 | Tampering | MITM between Cloudflare edge and phone | accept | TLS at Cloudflare edge; traffic from phone to CF edge is encrypted; only Cloudflare itself can observe. |
| T-CF-R-01 | Repudiation | No logs of who used the bearer token | mitigate | `data/errors.jsonl` captures failed auth attempts. `logError('lan.bearer.reject-origin', ...)` fires on origin mismatch. Cloudflare Zero Trust dashboard shows tunnel request logs. |
| T-CF-I-01 | Info Disclosure | Bearer token in URL query string | mitigate | All bearer tokens must be in `Authorization: Bearer` header, never URL params. Verified in existing `bearer.ts` tests. |
| T-CF-I-02 | Info Disclosure | Token stored in `.env` on disk | accept | `.env` is gitignored; `chmod 600` enforced by `ensureLanToken()` in `bearer.ts`. Single-user Mac. |
| T-CF-I-03 | Info Disclosure | Personal data transits Cloudflare | accept | Named-tunnel traffic attributed to `Arunever614@gmail.com` in CF Zero Trust dashboard. User accepted this at pivot decision. Personal notes, not business data. |
| T-CF-D-01 | Denial of Service | Rate limit saturation by runaway client | mitigate | 30/min per token, env-configurable (`BRAIN_LAN_RATE_LIMIT`). Single-user; runaway bug, not attacker. |
| T-CF-D-02 | Denial of Service | Tunnel goes down (Mac offline, cloudflared crash) | accept | Offline page renders in APK. User restarts `cloudflared tunnel run brain`. R-3 (service install) mitigates crash scenario. |
| T-CF-E-01 | Elevation of Privilege | Extension content-script injecting into popup | accept | Extension uses no content scripts. Service worker + popup are isolated MV3 contexts. |
| T-CF-E-02 | Elevation of Privilege | `ALLOWED_ORIGINS` bypass via `null` origin | mitigate | `bearer.ts` `validateOrigin()` rejects `null` origin. Already tested in `bearer.test.ts`. |

**Bearer rotation cadence:** no mandatory cadence for personal use. Rotate after: (a) suspecting device compromise, (b) sharing device with another person, (c) sharing screen during a session. `scripts/rotate-token.sh` or POST to `/api/settings/rotate-token`.

**Cloudflare WAF posture:** free tier has no WAF rules available. Rate limit (30/min per token) is the primary DoS protection. Named-tunnel IPs are obfuscated behind Cloudflare; the Mac's public IP is not exposed.
