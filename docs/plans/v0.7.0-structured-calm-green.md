# v0.7.0 — Structured Calm Green visual refresh

## 0. One-sentence goal

Swap the production visual system from the current indigo-on-slate palette
(`DESIGN.md`) to the editorial **emerald green / Newsreader / Inter** palette
specified in `DESIGN_STRUCTURED_CALM_GREEN.md`, by editing **values only** in
`src/styles/tokens.css` (variable names preserved per the §9.0 mapping in the
spec) and adding Newsreader to the font setup — without rewriting components
and without breaking any shipped page.

## 1. Scope and non-scope

### In scope (this phase)

| Tier | Items | Why included |
|------|-------|--------------|
| **GATE** | G-1..G-4 (close §0 gates in the spec) | The spec explicitly forbids §9 work until all four gates are green. |
| **T1 — token remap** | T-1 expand spec mapping · T-2 edit `tokens.css` light values · T-3 add 6 new M3-only tokens · T-4 add Newsreader font wiring | Core swap; isolated to two files |
| **T1 — dark theme** | T-5 finalise dark-theme palette · T-6 contrast-pass dark theme · T-7 wire dark values into `tokens.css` `[data-theme="dark"]` | Gate (b) deliverable; cannot ship without |
| **T2 — component audit** | T-8 grep hardcoded hex/colors · T-9 grep hardcoded font literals · T-10 fix any hits | Preserves the "values-only" assumption; surface drift is the highest-blast-radius risk |
| **T2 — visual sweep** | T-11..T-19 page-by-page sweep (`/`, `/ask`, `/inbox`, `/search`, `/settings`, `/setup`, `/unlock`, `/capture`, `/items/[id]`) | One task per page, with screenshot before/after |
| **T2 — accessibility** | T-20 keyboard-focus pass · T-21 reduced-motion pass · T-22 a11y axe scan | Non-negotiable; spec §5.3 requires 2px primary focus ring |
| **T3 — release** | T-23 design-doc cross-references · T-24 `DESIGN.md` archive note · T-25 smoke + version bump (→ 0.7.0) + tag | Standard release wave |

### Explicitly out of scope (phase backlog or future)

- **Tertiary (rose) wiring into components.** Spec §2.1 marks `tertiary` and `tertiary-container` as **Reserved — unused until SRS lands (v0.8.0)**. Set the values in `tokens.css` but do NOT use them in any component. SRS will wire them in v0.8.0.
- **Citation-chip and GenLink visual updates beyond token swap** — both are spec-defined in §7.4 / §7.5; the *colors* update via the token swap automatically. No layout/markup changes.
- **Dark-mode toggle UX** — already shipped in v0.1.0; this phase only updates the palette, not the toggle itself.
- **Mobile-only layout changes** — the spec's §3.2 mobile rules are type-size-only; no breakpoint changes, no bottom-nav rework.
- **Component refactors** — no new components, no removals. Same DOM, new pixels.
- **Marketing site / landing page updates** — Brain has none; not applicable.
- **Brain logo / favicon** — out of scope; visual identity is text-driven only.

## 5. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| Hardcoded color in a rarely-used component (e.g., `OfflineIndicator`) ignores token swap | T-8 grep + T-10 fix; visual sweep also catches anything that survives grep |
| Newsreader font causes FOUT on slow connections | `next/font` self-hosts (no CDN dep); use `display: swap` consistent with existing Inter setup |
| Dark theme finalisation produces values that fail AA in unexpected pairings (e.g., chip-on-card) | T-6 contrast pass is iterative; budget extra time |
| User dislikes the green after seeing it on real content | T-1 BEFORE screenshots + T-11..T-19 AFTER screenshots make rollback easy: `git revert` two commits (T-2 + T-7) |
| Rolling out swap during an active phase causes user disorientation | This phase is its own release. Do NOT bundle with v0.6.2 backup or anything else. |
| Spec drifts (someone edits `DESIGN_STRUCTURED_CALM_GREEN.md` after T-2 ships) | Pin to commit SHA at T-2 start in the running-log entry |
| Citation chips on dark theme become illegible | Specifically contrast-check `primary` text on `secondary-container` in dark mode (T-6) |
| `--font-article` rename / repurpose breaks existing reading-mode CSS | T-9 grep catches; T-19 visual sweep on `/items/[id]` confirms |

## 7. What "complete" means here

1. Spec `DESIGN_STRUCTURED_CALM_GREEN.md` status moves `PROPOSED` → `ADOPTED`.
2. `src/styles/tokens.css` reflects the spec (light + dark) with names preserved.
3. Newsreader is loaded via `next/font/google` and wired to `--font-article`.
4. The 9 sweep pages render the new palette in both themes without visual regressions.
5. Accessibility (focus, motion, axe) clean.
6. `DESIGN.md` and `DESIGN_SYSTEM.md` are either archived (G-4=replace) or annotated as superseded.
7. Tag `v0.7.0` shipped to `main`; trackers updated; RUNNING_LOG entry written.

**What "complete" does NOT mean:**
- It does NOT mean every component has been redesigned — same DOM, new pixels.
- It does NOT mean SRS / GenLink / GenPage features are themed (those land with their own phases).
- It does NOT mean the design has been validated with a second user — Brain is single-user.
- It does NOT mean dark-mode is the default — toggle behaviour unchanged.
