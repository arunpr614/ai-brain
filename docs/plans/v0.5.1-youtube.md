## 1. Goal

Make YouTube URLs capturable across the Chrome extension and the Android APK — same share target, same server endpoint, same library retrieval. Today they fail `422 capture_failed` because Readability can't extract JS-rendered pages. After v0.5.1, sharing a YouTube URL produces a library item with:

- **Title** = raw video title (no mangling; enrichment may rewrite to remove click-bait)
- **Author** = channel name (persists regardless of title rewrite; enrichment sees it via §5.2.1)
- **Body** = pure timestamped transcript `[MM:SS] spoken line` — feeds the existing chunker, FTS5, and vector pipelines with zero structural churn
- **Source URL** normalized to canonical `https://www.youtube.com/watch?v=<id>` so `youtu.be/X` and `youtube.com/watch?v=X` dedup against each other
- **`duration_seconds`** populated so the library UI can show "1h 23m" next to video items

**Shipping principle:** one new file, one new migration, **zero new dependencies**, zero changes to the extension or APK source trees. Two small internal changes in `src/lib/enrich/prompts.ts` (C-1 absorbed) and a tiny badge tweak in library item display (TB-3 absorbed). All external-facing logic lives server-side so the two existing clients get this feature for free.

**Inline InnerTube + XML parser** (replaces the `youtube-transcript` dep — C-2):

```ts
const INNERTUBE_URL = "https://www.youtube.com/youtubei/v1/player";
const INNERTUBE_BODY = (videoId: string) => ({
  videoId,
  context: {
    client: {
      clientName: "ANDROID",        // per research §1; current workaround
      clientVersion: "20.10.38",    // pin to known-working version
      hl: "en",
      gl: "US",
    },
  },
});

async function fetchInnerTubePlayer(videoId: string, signal?: AbortSignal) {
  const res = await fetch(INNERTUBE_URL, {
    method: "POST",
    headers: { "content-type": "application/json" },
    body: JSON.stringify(INNERTUBE_BODY(videoId)),
    signal,
  });
  if (!res.ok) throw new YoutubeCaptureError("fetch_failed", `InnerTube returned ${res.status}`);
  return res.json();
}

// Parse YouTube timed-text XML (srv1/default format):
// <transcript>
//   <text start="0" dur="2.8">We're no strangers to love</text>
//   ...
// </transcript>
function parseTimedTextXml(xml: string): TranscriptSegment[] {
  // jsdom is already a runtime dep (used by Readability in url.ts);
  // no new dep required. DOMParser avoids regex quirks around escaped
  // angle brackets inside text content.
  const dom = new JSDOM(xml, { contentType: "text/xml" });
  const nodes = dom.window.document.querySelectorAll("text");
  const segments: TranscriptSegment[] = [];
  nodes.forEach((node) => {
    const start = parseFloat(node.getAttribute("start") ?? "0");
    const dur   = parseFloat(node.getAttribute("dur") ?? "0");
    // textContent handles HTML-entity decoding (&#39; → ', &amp; → &)
    const text = (node.textContent ?? "").trim();
    if (text) segments.push({ offset: start * 1000, duration: dur * 1000, text });
  });
  return segments;
}

interface TranscriptSegment { offset: number; duration: number; text: string; }
```

**Main extractor flow:**

```ts
export async function extractYoutubeVideo(videoId: string, originalUrl: string): Promise<YoutubeExtracted> {
  const player = await fetchInnerTubePlayer(videoId);

  // Availability check
  if (!player.videoDetails) {
    throw new YoutubeCaptureError("video_unavailable", "Video is private, deleted, or unavailable.");
  }
  if (player.videoDetails.isLive || player.videoDetails.isLiveContent && !player.captions) {
    throw new YoutubeCaptureError("live_stream", "Live stream — transcript not available yet.");
  }

  const title = String(player.videoDetails.title ?? videoId);
  const author = String(player.videoDetails.author ?? "") || null;
  const duration_seconds = parseInt(player.videoDetails.lengthSeconds ?? "0", 10) || null;

  const captionTracks = player.captions?.playerCaptionsTracklistRenderer?.captionTracks;
  if (!Array.isArray(captionTracks) || captionTracks.length === 0) {
    return {
      title, author, duration_seconds,
      source_url: canonicalYoutubeUrl(videoId),
      body: "[No transcript available for this video]",
      extraction_warning: "no_transcript",
    };
  }

  const baseUrl = captionTracks[0].baseUrl;
  const xmlRes = await fetch(baseUrl);
  if (!xmlRes.ok) throw new YoutubeCaptureError("fetch_failed", `Timed-text fetch returned ${xmlRes.status}`);
  const xml = await xmlRes.text();
  const segments = parseTimedTextXml(xml);

  if (segments.length === 0) {
    // Guard for captionTracks present but XML empty (edge case) — L-2 absorbed
    return {
      title, author, duration_seconds,
      source_url: canonicalYoutubeUrl(videoId),
      body: "[No transcript available for this video]",
      extraction_warning: "no_transcript",
    };
  }

  const MAX_SEGMENTS = 7200;
  const truncated = segments.length > MAX_SEGMENTS;
  const body = formatTranscriptBody(truncated ? segments.slice(0, MAX_SEGMENTS) : segments);

  return {
    title, author, duration_seconds,
    source_url: canonicalYoutubeUrl(videoId),
    body,
    extraction_warning: truncated ? "transcript_truncated_2h" : null,
  };
}
```

**Body formatting** — pure transcript, no metadata header (C-1 absorbed):

### 5.2.1 Enrichment context via title composition (C-1 absorbed)

The enrichment pipeline (`src/lib/queue/enrichment-worker.ts`) passes `item.title` and a 12,000-char-truncated `item.body` to the LLM. For a 2-hour transcript, the body truncation loses ~99% of the video. For articles this is fine (articles have their key point in the intro); for videos the opening 3 minutes is usually announcements, not content.

**Resolution:** modify the enrichment-worker call site, NOT the stored body:

```ts
// src/lib/queue/enrichment-worker.ts (existing load → pass through)
const { title, body, source_type, author, duration_seconds } = item;

const effectiveTitle = source_type === "youtube" && author
  ? `${title} — ${author}${duration_seconds ? ` (${formatDurationForTitle(duration_seconds)})` : ""}`
  : title;

const userPrompt = enrichmentUserPrompt({ source_type, title: effectiveTitle, body });
```

`formatDurationForTitle` is a tiny helper (5 lines, `${H}h${M}m` or `${M}m`) co-located with the enrichment worker.

The effect: the LLM sees `"Growth loops primer — Lenny's Podcast (1h 23m)"` as the title, giving it channel + duration context for summary, category, and tags. The stored `title` column stays clean (just `"Growth loops primer"`), so the library UI shows title + author separately without double-printing.

This resolves H-3 without polluting `body` (and therefore without polluting retrieval chunks).

### 5.4 Transcript → body formatting

```ts
const WINDOW_MS = 30_000;

function formatTranscriptBody(segments: TranscriptSegment[]): string {
  let windowStart = 0;
  const paragraphs: string[] = [];
  let lines: string[] = [];
  for (const seg of segments) {
    if (seg.offset - windowStart > WINDOW_MS && lines.length > 0) {
      paragraphs.push(lines.join("\n"));
      lines = [];
      windowStart = seg.offset;
    }
    lines.push(`[${msToTimestamp(seg.offset)}] ${seg.text.trim()}`);
  }
  if (lines.length) paragraphs.push(lines.join("\n"));
  return paragraphs.join("\n\n");
}

function msToTimestamp(ms: number): string {
  const s = Math.floor(ms / 1000);
  const m = Math.floor(s / 60);
  const h = Math.floor(m / 60);
  return h > 0
    ? `${h}:${String(m % 60).padStart(2, "0")}:${String(s % 60).padStart(2, "0")}`
    : `${m}:${String(s % 60).padStart(2, "0")}`;
}
```

### 5.6 Telemetry

Skip the "success event" log (L-3 absorbed). The `items` row IS the success record. On failure, existing `share.http.capture-failed` shape covers it with one new `code` enum value `"youtube_<sub>"` where sub is `no_captions`, `video_unavailable`, `live_stream`, or `fetch_failed`.

### 5.7 Ask / retrieval integration (zero changes to retrieval)

The existing chunker, embedding pipeline, and retriever all operate on `items.body`. Because the body is pure timestamped transcript (C-1 resolution), chunks are clean transcript content, not metadata. Citation chips will land on real spoken content.

- Chunker finds paragraph boundaries at 30-second gaps → chunks align to ~30-second spoken windows
- FTS5 tokenizes transcript text → full-text search works
- Vector embedding via `nomic-embed-text` → semantic retrieval works
- Ask UI renders `[CITE:<chunk_id>]` → clicking opens the item page at the chunk. Time-anchored deep-linking (jump to `&t=272s`) deferred to v0.5.2.

## 9. Rollback

Plan uses one additive migration (`007_youtube_duration.sql`) and one isolated new file (`youtube.ts`).

**Soft rollback** (feature broken, data safe): `git revert` the T-YT-6 commit. Extractor file remains; dispatch branch disappears; YouTube URLs return to 422.

**Hard rollback** (unlikely): `scripts/restore-from-backup.sh` → pre-v0.5.1 SQLite snapshot. `duration_seconds` column silently survives the restore as null on old rows; no orphan data.

**YouTube breaks our inline parser** (likeliest failure in 6–12 months): open `src/lib/capture/youtube.ts`. Options, in increasing cost:
1. Update `clientName`/`clientVersion` to current Android app version (5 min)
2. Re-introduce `youtube-transcript@~1.x` as a fallback behind a feature flag (30 min)
3. Rewrite extraction against a different backend (1–2 h)
