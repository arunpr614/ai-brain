## 0. Goal of v0.6.0

Move the Brain backend from Mac-hosted Ollama+Next.js to a cloud-hosted Hetzner VM. Keep the same `brain.arunp.in` URL, same SQLite schema, same APK/extension clients, same bearer auth. Swap Ollama → Anthropic (Haiku batch enrichment + Sonnet realtime Ask) + Gemini embeddings.

**The architectural insight added in this revision:** every AI vendor is replaceable behind a thin provider-agnostic interface. Anthropic-direct is the **default** because it gives Batch API (50% off) and is the highest-quality option. **OpenRouter is wired in as a swap target** so a future cost-driven or quality-driven model change is a one-env-var swap, not a refactor.

## 1. User-locked architectural decisions

These are non-negotiable and shape every task:

1. **Always-on architecture.** Mac sleeping = Brain unreachable is the pain point we're solving. (locked 2026-05-12)
2. **No home server, no Mac Mini.** Cloud VM only. (locked 2026-05-12)
3. **Cloud host: Hetzner CX23 Helsinki, $5.59/mo.** (locked 2026-05-13 after Hetzner CAX11 sold out + Lightsail Mumbai $10 rejected)
4. **Enrichment timing: daily batch is default; manual "Enrich now" button is the escape hatch.** (locked 2026-05-12)
5. **AI provider quality-first:** Anthropic Haiku 4.5 (enrichment, batch) + Anthropic Sonnet 4.6 (Ask, realtime streaming). (locked 2026-05-12)
6. **Embeddings: Gemini text-embedding-004 (free tier, 768-dim).** Matches existing `chunks_vec` schema; zero re-embed required. (locked 2026-05-12)
7. **Provider-agnostic LLM wrapper.** Architecture must allow swap to OpenRouter (and any model OR exposes) via env var with **zero code change** beyond config. (locked 2026-05-14, this revision)
8. **OpenRouter is standby, not primary.** Reason: Anthropic Batch API (50% off enrichment) is not exposed via OpenRouter. (locked 2026-05-14)
9. **Privacy bar:** paid-tier no-training-on-inputs. Bearer auth unchanged. No multi-user. (locked since v0.5.0)
10. **Cutover window:** 03:00 IST (when user is asleep). Total downtime budget: 30 minutes. (per S-7 runbook)

## 3. Architecture overview

### 3.1. Provider-agnostic LLM wrapper

**v1.1 update:** the structure below reflects the **shipped** B-1..B-7 layout. v1.0 sketched a different shape; this is what `phase-b/v0.6.0` actually ships.

```
src/lib/llm/
├── types.ts          # LLMProvider, GenerateOptions, GenerateResult, etc.
├── errors.ts         # LLMError class (kept separate from types per type-only-file convention)
├── factory.ts        # getEnrichProvider() + getAskProvider() env-driven, memoized
├── ollama.ts         # OllamaProvider class + back-compat module-level wrappers
├── anthropic.ts      # AnthropicProvider — fetch only, no SDK
├── openrouter.ts     # OpenRouterProvider — fetch only, no SDK
└── *.test.ts         # 24 tests across the wrappers
```

**Interface (as shipped in B-1):**

```ts
export interface GenerateMetrics {
  // Honest 3-field schema — only fields every provider can populate truthfully.
  // Ollama-specific timings (load_duration, prompt_eval_tps) live on
  // src/lib/llm/ollama.ts:getLastOllamaDiagnostics() instead of riding on every result.
  input_tokens: number;
  output_tokens: number;
  wall_ms: number;
}

export interface LLMProvider {
  /** Realtime non-streaming generate; used internally by generateJson and by submitBatch. */
  generate(opts: GenerateOptions): Promise<GenerateResult>;

  /** Realtime streaming; used by Ask SSE handler. AsyncIterable, not AsyncGenerator —
   *  call sites only need `for await`; AsyncGenerator was over-narrow. */
  generateStream(opts: GenerateStreamOptions): AsyncIterable<string>;

  /** Returns parsed JSON; retries once at lower temperature on parse failure;
   *  throws LLMError("invalid_response") with raw on cause if both fail. */
  generateJson<T = unknown>(opts: GenerateOptions): Promise<GenerateJsonResult<T>>;

  /** Cheap reachability probe used by /api/ask, /api/search, enrichment-worker. */
  isAlive(): Promise<boolean>;

  /** Optional batch capability. Anthropic implements; OpenRouter omits.
   *  Phase C gates on `typeof provider.submitBatch === "function"`. */
  submitBatch?(requests: unknown[]): Promise<{ batch_id: string }>;
  pollBatch?(batch_id: string): Promise<unknown>;
}
```

**Concrete batch shapes** (Anthropic-only, B-4): `AnthropicBatchRequest`, `AnthropicBatchPoll`, `AnthropicBatchResultEntry` are exported from `src/lib/llm/anthropic.ts`. v1.0 said "shapes defined in B-4 when the concrete call site exists" — they now exist; this revision points there instead of inlining duplicate definitions.

**Why fetch-only (no SDK):** symmetry with OpenRouter (which is OpenAI-compatible and trivially fetch-able), zero transitive vuln surface (`@anthropic-ai/sdk@0.96.0` brought 4 vulns: 3 moderate + 1 high), no caret-pin SDK that can shift SSE shapes between minor bumps. Pre-verified by spike **S-10** against the live API at $0.000861.

**Selection:**

```ts
// src/lib/llm/factory.ts
export function getEnrichProvider(): LLMProvider {
  switch (process.env.LLM_ENRICH_PROVIDER ?? "anthropic") {
    case "anthropic": return anthropicProvider({ model: process.env.LLM_ENRICH_MODEL ?? "claude-haiku-4-5-20251001" });
    case "openrouter": return openrouterProvider({ model: process.env.LLM_ENRICH_MODEL ?? "anthropic/claude-haiku-4-5" });
    case "ollama": return ollamaProvider({ model: process.env.LLM_ENRICH_MODEL ?? "qwen2.5:7b" });
  }
}
export function getAskProvider(): LLMProvider { /* same shape, different env vars */ }
```

**Env var contract (locked):**

```
# Default v0.6.0 wiring
LLM_ENRICH_PROVIDER=anthropic        # anthropic | openrouter | ollama
LLM_ENRICH_MODEL=claude-haiku-4-5-20251001
LLM_ASK_PROVIDER=anthropic
LLM_ASK_MODEL=claude-sonnet-4-6
LLM_ENRICH_BATCH=true                # use Batch API; only honored when provider supports it

ANTHROPIC_API_KEY=sk-ant-...         # required when provider=anthropic
OPENROUTER_API_KEY=sk-or-...         # required when provider=openrouter

# Embedding (separate; no OR coverage needed at this volume)
EMBED_PROVIDER=gemini                # gemini | ollama (fallback)
EMBED_MODEL=text-embedding-004
GEMINI_API_KEY=...
```

**OpenRouter request hardening (when used):** every request body must include the privacy/pinning block from `docs/research/openrouter-provider-evaluation.md`:

```json
{
  "model": "anthropic/claude-sonnet-4-6",
  "provider": {
    "order": ["Anthropic"],
    "allow_fallbacks": false,
    "data_collection": "deny"
  }
}
```

This is not optional — without it OR can route to upstreams that log inputs. Tests must enforce it.

### 3.2. Embedding wrapper

Same pattern, simpler:

```
src/lib/embed/
├── types.ts          # EmbedProvider interface
├── factory.ts        # selectProvider()
├── gemini.ts         # default (free tier, 768-dim)
└── ollama.ts         # fallback for local dev
```

Output dimension is locked at 768 to match `chunks_vec float[768]`. If a future provider only emits 1024/1536, the wrapper must reject — no silent dim mismatch.

### 3.3. Daily batch job

`node-cron` inside the Next.js process (no systemd timer). Fires at `0 3 * * *` UTC.

```
src/lib/queue/
├── enrichment-worker.ts          # existing realtime path; refactored to use factory
└── enrichment-batch.ts           # new — daily Batch API submitter + poller
```

**Flow:**
1. 03:00 UTC — sweep all `enrichment_jobs WHERE state='pending'`.
2. Build one Anthropic Message Batch with N requests.
3. Submit → store `batch_id` on each `items` row + `enrichment_state='batched'`.
4. Poll every 5 min until `processing_status='ended'` or 24h timeout.
5. Write results back; mark `done` or `error`.
6. On error: retry next day (max 3 attempts via `enrichment_jobs.attempts`).

The poll loop uses `setInterval`, NOT a separate cron, because the batch must be polled by the same process that submitted it (no shared state otherwise).

### 3.4. Manual "Enrich now" endpoint

`POST /api/items/:id/enrich` — already exists for realtime Ollama. Refactor it to:
1. Use `getEnrichProvider()` instead of hard-coded Ollama call.
2. If `LLM_ENRICH_BATCH=true` AND item is `pending`, this endpoint forces realtime path (bypasses batch).
3. Sets `items.enrichment_state='running'` → `'done'|'error'`.

UI change: the "⏳ Enriching…" pill gains a "Queued for tonight's batch — Enrich now?" affordance. Lane L will polish; Lane C ships the data + endpoint.

### 3.5. Backup to Backblaze B2

Per S-7 runbook:
- `cron` (system, not node-cron): every 6 hours.
- Script: `sqlite3 .backup` → `gzip` → `gpg --symmetric` → `rclone` to B2 bucket.
- gpg passphrase lives **on user's Mac**, not on the server (verifiable: server can write but cannot read its own backups without the key).
- Lifecycle rule on B2: delete objects older than 30 days.

### 3.6. Tunnel migration

Same Cloudflare Named Tunnel `brain.arunp.in`:
1. Provision tunnel credentials on Hetzner box.
2. Test in parallel (different hostname during preview).
3. At cutover: stop Mac cloudflared → start Hetzner cloudflared → CF DNS auto-updates.
4. Mac Brain (`launchctl unload`) is the last step.
