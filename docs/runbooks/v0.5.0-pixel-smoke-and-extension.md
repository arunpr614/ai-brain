# Runbook 1 — T-CF-14 Physical Pixel Smoke over Cellular

## 1.1 Goal

Prove the APK works on your real Pixel over **cellular data** (not Wi-Fi), so when you're away from home Brain still works.

## 1.2 Pass criteria

From your Pixel on cellular, you can:

1. Share a URL from Chrome → appears in Mac Library within 5 seconds
2. Share a PDF → captured with searchable extracted text
3. Ask a >90-second question → answer streams word-by-word to completion
4. Airplane mode → see Brain's styled offline page (not a raw browser error)

## 1.3 Time budget

~30-40 minutes including one-time phone setup.

## 1.4 What you'll need

- Your Pixel phone
- A USB-C cable (one-time; only for the initial APK install)
- The Mac awake; dev server running; tunnel running; Ollama running
- Your Brain PIN

## 1.5 Phase A — Verify the Mac is ready (5 min)

Open a new Terminal window. Run these one at a time; PASS criteria below each.

### A1. Tunnel status

```bash
curl http://127.0.0.1:20241/ready
```

- ✅ PASS: `{"status":200,"readyConnections":4,"connectorId":"..."}`
- ❌ FAIL: "connection refused" → tunnel died. Tell me; I'll restart it.

### A2. Tunnel actually reaches Brain from the outside

```bash
curl https://brain.arunp.in/api/health
```

- ✅ PASS: `{"ok":true,"ts":...}` OR `{"error":"unauthenticated"}` — both mean Cloudflare is routing to the Mac
- ❌ FAIL: "Could not resolve host" or timeout → DNS/tunnel problem. **Stop.**

### A3. Ollama is running

```bash
curl http://localhost:11434/api/tags
```

- ✅ PASS: JSON blob listing models. Verify you see `qwen2.5:7b-instruct-q4_K_M` + `nomic-embed-text`.
- ❌ FAIL: "connection refused" → Ollama isn't running. Open Ollama from Applications, wait 5 seconds, re-run. If still failing, tell me.

## 1.6 Phase B — One-time Pixel developer setup

Skip if already done previously.

### B1. Enable developer options

1. On Pixel: **Settings → About phone → Build number**. Tap **Build number 7 times**. A "You are now a developer" toast appears.
2. Back out: **Settings → System → Developer options**. Turn **USB debugging** ON.

You won't touch these again.

## 1.7 Phase C — Install the APK (5 min; USB required once)

### C1. Connect USB

Plug Pixel → Mac via USB-C.

### C2. Authorize debugging

On Pixel: tap **Allow** (check "Always allow from this computer") on the USB-debug pop-up.

### C3. Confirm Mac sees the phone

```bash
cd /Users/arun.prakash/Documents/GitHub/arun-cursor/Initiatives/Arun_AI_Projects/ai-brain
adb devices
```

- ✅ PASS: Pixel listed with serial number + `device` suffix
- ❌ FAIL: empty list OR `unauthorized` → re-plug USB, tap Allow again, retry

### C4. Install APK

```bash
adb install -r data/artifacts/brain-debug-0.4.0.apk
```

- ✅ PASS: `Success` after ~10 seconds
- ❌ FAIL: `INSTALL_FAILED_UPDATE_INCOMPATIBLE` → older Brain APK with different keystore. Run `adb uninstall com.arunprakash.brain` then retry `adb install`.
- ❌ FAIL: other → tell me the full error.

### C5. Unplug

Unplug USB. From here on, the phone uses cellular.

## 1.8 Phase D — Force cellular (2 min)

This is critical — if the test accidentally runs on Wi-Fi, we learn nothing.

### D1. Turn off Wi-Fi

Pull down quick-settings shade twice. Tap **Wi-Fi** tile OFF. Leave **Mobile data / Cellular** ON.

### D2. Sanity-check cellular works

Open Chrome mobile. Visit `https://google.com`. Page loads → cellular is alive.

### D3. Verify cellular signal

Confirm a signal bar at the top of the Pixel screen before proceeding.

## 1.9 Phase E — First-run pairing (5 min)

### E1. Launch APK

Find **Brain** in the app drawer. Tap.

### E2. Unlock

Enter your Brain PIN (same one as the Mac web app).

### E3. QR scanner loads

The app routes to the camera view at `/setup-apk`.

### E4. Show QR on Mac

On Mac browser, open `https://brain.arunp.in/settings/lan-info`. (If prompted, unlock with PIN.) You'll see a QR code.

### E5. Scan

Hold Pixel up to Mac screen so the camera sees the QR.

- ✅ PASS: APK shows "Testing connection..." → "Paired — redirecting..." → lands on Library
- ❌ FAIL: "Could not reach Brain, Cloudflare tunnel must be up" → tunnel problem. **Stop**, tell me.
- ❌ FAIL: "QR url deprecated" → stale QR. Reload the Mac pairing page (regenerates QR on each request) and rescan.

## 1.10 Phase F — Core smoke tests over cellular (~15 min)

### F1. Share a URL

1. Chrome mobile → any article (e.g. `https://en.wikipedia.org/wiki/Cloudflare`)
2. Tap ⋮ menu → **Share...**
3. Pick **Brain** from the share sheet
4. APK flashes "Capturing..." → navigates to the new item

- ✅ PASS: on Mac, refresh Library; article appears within 5 s
- ❌ FAIL: note what the APK showed + check `data/errors.jsonl` on Mac; tell me

### F2. Share a PDF

1. On Pixel, open any PDF (Gmail attachment, Drive file, Files app — any)
2. Share → **Brain**
3. APK may briefly show a progress indicator

- ✅ PASS: PDF appears in Mac Library with searchable extracted text
- ❌ FAIL: tell me the error

### F3. Long-running Ask (>90 s) — the keepalive validation

This test validates the 10-minute Cloudflare tunnel keepalive. Default is 90 s; if our config didn't take, the stream cuts off at ~90 s.

1. In APK, navigate to **Ask** tab (or `/ask`)
2. Type exactly:
   > *Write a 500-word essay on the history of the internet, covering ARPANET, TCP/IP, the World Wide Web, and the smartphone era. Be thorough.*
3. Tap Send
4. Watch words stream in (word-by-word or sentence-by-sentence)
5. Expected duration: ~2-5 min on M1 Pro

- ✅ PASS: full answer completes; maybe shows citations below
- ❌ FAIL — stream cuts off around 90 s with "transfer closed" or similar: **keepalive config broken**. Important; tell me.
- ❌ FAIL — "Ollama offline" / 503: Ollama crashed or model not loaded. On Mac: `ollama run qwen2.5:7b-instruct-q4_K_M` once to reload.

### F4. Offline behavior

1. Pixel: turn on **Airplane mode** (quick-settings → airplane icon)
2. In APK: tap any link or refresh

- ✅ PASS: Brain-styled "You're offline" page
- ❌ FAIL: generic Android network error → tell me

3. Turn off airplane mode; wait 5 s for reconnect
4. Tap something in the APK

- ✅ PASS: loads normally again

## 2.2 T-CF-15 — Chrome pre-flight smoke (your job, 5 min)

### 2.2.1 Why this exists

Before I build an extension that fetches from `https://brain.arunp.in`, we need to verify that Chrome 147 service workers can actually fetch an external HTTPS origin. If they can't, we need workarounds. Let's prove it now, not after 500 lines of code.

### 2.2.2 Steps

#### G1. Note Chrome version

1. Chrome → `chrome://version/` — note the version number on the top line. Tell me what you see.

#### G2. Open DevTools

1. Open any tab
2. ⌘+Option+I (or right-click → Inspect)

#### G3. Console test

Click the **Console** tab. Paste this exact command; press Enter:

```javascript
fetch('https://brain.arunp.in/api/health').then(r => r.status).then(s => console.log('status:', s)).catch(e => console.error('error:', e))
```

- ✅ PASS: `status: 401` (401 is correct — we didn't send a bearer token; the point is the request *reached* the server)
- ❌ FAIL: `error: TypeError: Failed to fetch` → CORS/network block. Tell me the full error; we may need `ALLOWED_ORIGINS` adjustment or a different approach.

#### G4. Report back

Tell me: Chrome version + G3 result. If PASS, I start T-CF-16 immediately. If FAIL, we debug.

# Appendix A — What to do if a step fails

Two rules:

1. **Don't guess at fixes.** Tell me the exact error; I've seen most of them before and can tell you the 10-second fix vs. the "needs real investigation" response.
2. **Don't proceed past a FAIL.** A test passing *after* an upstream step failed is not a real pass — it's a false signal.

## A.1 Common failures

| Symptom | Likely cause | Fix |
|---|---|---|
| `connection refused` on `127.0.0.1:20241` | Tunnel process died | Restart: `cloudflared tunnel run brain` |
| `Could not resolve host brain.arunp.in` | DNS propagation issue or tunnel not routing | Check Cloudflare dashboard → Zero Trust → Tunnels |
| `INSTALL_FAILED_UPDATE_INCOMPATIBLE` | Old APK with different keystore signature | `adb uninstall com.arunprakash.brain` then retry |
| APK pairing: "QR url deprecated" | Stale QR from a previous pairing session | Reload Mac pairing page to regenerate |
| APK pairing: "Could not reach Brain" | Tunnel down OR outbound blocked | Retry A1 / A2 |
| Ask stream cuts off at ~90 s | Cloudflare keepalive config not applied | **Tell me** — this is a real bug, not user error |
| Chrome `fetch` returns `TypeError: Failed to fetch` | CORS or extension permission mismatch | **Tell me** — I'll adjust `ALLOWED_ORIGINS` or extension manifest |

## A.2 Always-safe recovery commands

```bash
# Restart tunnel (from Mac Terminal)
cloudflared tunnel run brain

# Restart Next.js dev server
cd /Users/arun.prakash/Documents/GitHub/arun-cursor/Initiatives/Arun_AI_Projects/ai-brain
```
