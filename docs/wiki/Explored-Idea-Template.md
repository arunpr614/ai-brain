# Explored Idea Template

Purpose: Prevent proposals and prototypes from being mistaken for implemented product behavior.
Audience: Product/design/engineering contributors and AI agents.
Verified against: `ea7b159515fc37f76ffdb83dedf2d33d17f9a193`.
Runtime evidence through: Not applicable unless a partial current implementation is separately cited.
Last reviewed: 2026-07-11.
Owner: AI Brain maintainer.

1. Idea name and status: Explored, Planned, Deferred, Rejected, Superseded, or Unknown.
2. Summary and user problem/opportunity.
3. Proposed behavior.
4. Source documents and prototype/design evidence.
5. Current implementation evidence, if partial.
6. Why it is not classified as Implemented.
7. Dependencies, risks and blockers.
8. Decision history and relationship to current features.
9. Documented next step only when a source provides one.
10. Confidence and last verification date.

Always link the living Feature Catalog. “Approved” or “current” within a dated planning package does not establish present implementation or deployment.
