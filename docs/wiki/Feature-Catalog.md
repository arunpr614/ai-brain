## Coverage Rules

Every in-scope route, action, migration, domain module, client integration, deployment artifact, and package script is classified in the tracked source inventory or command registry. An apparent omission must be recorded as internal, supporting, deprecated, planned, or excluded with rationale.

When adding or changing a feature:

1. Update the source inventory.
2. Update this catalog and the tracked feature ledger.
3. Update command safety for new scripts.
4. Cite the exact baseline and verification evidence.
5. Keep runtime Unknown unless dated evidence proves it.
6. Republish through [Documentation Maintenance](Documentation-Maintenance).
