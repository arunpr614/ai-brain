# Glossary

Purpose: Define project-specific terms, aliases, and commonly confused concepts.
Audience: All wiki readers.
Verified against: `23868faf13c8e3d0821715e6f5d0e3d2af1e1a34`.
Runtime evidence through: Not applicable.
Last reviewed: 2026-07-11.
Owner: AI Brain maintainer.

| Term | Definition |
|---|---|
| AI Brain | Repository/project name; parts of the current UI use the alias AI Memory |
| Item | Canonical library record; historical material may call it a card or save |
| Source | Captured/imported material associated with an item |
| Capture artifact | Sanitized retained extraction/provenance evidence referenced by metadata |
| Standalone note | A note-source item created through Capture |
| My notes / attached note | One private Markdown document attached to an existing item |
| Original | Saved source body, separate from Digest and My notes |
| Digest | AI-derived summary/output |
| Capture source | Ingestion channel such as web, Android, extension, Telegram, or Recall |
| Source type/platform | Content kind and extractor/platform identity; distinct from capture source |
| Quality | Source fidelity/readiness state, not simply saved/failed |
| Review | Attention inbox for source quality/problems, not spaced repetition |
| Needs Upgrade | Queue focused on weak or insufficient source text |
| Source-reading Focus | Minimal item reading using `mode=focus` |
| Note Focus | Full-viewport attached-note editor using `note_mode=focus` |
| Ask | Product surface for cited retrieval-augmented questions |
| Related/connections | Query-time semantic similarity, not a persisted graph |
| FTS/semantic/hybrid | Lexical, vector, and reciprocal-rank-fused retrieval modes |
| Feature Council | Dated research/planning corpus; not implementation proof |
