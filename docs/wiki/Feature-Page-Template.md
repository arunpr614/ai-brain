# Feature Page Template

Purpose: Provide the required structure for significant implemented or partial feature documentation.
Audience: Documentation contributors and AI agents.
Verified against: `8c1341100b174fe4ca518e6a745c30b9078df21c`.
Runtime evidence through: Not applicable; this is a template.
Last reviewed: 2026-07-11.
Owner: AI Brain maintainer.

1. Feature name.
2. Current status, availability, confidence, current-main baseline, runtime evidence and verification date.
3. Summary, user problem and target users.
4. Entry points and user journey.
5. Detailed functionality and covers/does-not-cover boundary.
6. Empty, loading, success and failure states.
7. Architecture, runtime flow and data flow.
8. Data model/storage and APIs/integrations.
9. AI/model behavior where applicable.
10. Authentication, authorization, security and privacy.
11. Configuration and feature flags.
12. Relevant files/modules and change-impact guidance.
13. Tests and verification.
14. Operational considerations and known limitations.
15. Related current features and explored ideas.
16. Pinned evidence references.

Use `Not applicable — reason` for a required section that does not apply. On family pages, give materially different sub-capabilities separate status and verification rows.
