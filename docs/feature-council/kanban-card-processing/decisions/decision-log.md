## Status vocabulary

- **Approved fact:** already established by stakeholder design acceptance, current repository, or the execution goal.
- **Accepted for execution:** council resolution to implement unless superseded by stronger evidence or explicit direction.
- **Assumption:** requires verification.
- **Deferred:** outside first-release scope.
- **Open validation:** default exists, but evidence can reopen it.
- **Review gate:** decided in product v2 but not yet cleared for implementation until UX/UI v2 and technical-plan v2 explicitly match.
