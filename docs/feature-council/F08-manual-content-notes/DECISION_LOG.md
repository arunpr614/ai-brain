# F08 Decision Log

| ID | Date | Decision | Status | Reason |
|---|---|---|---|---|
| D01 | 2026-07-10 | Model this feature as one attached note document per existing item, not a new item/card. | Accepted | Preserves the user's mental model and avoids duplicate cards, graph nodes, enrichment, and deletion ambiguity. |
| D02 | 2026-07-10 | Persist manual notes separately from original content and AI output. | Accepted | Authorship, privacy, search provenance, and lifecycle must remain distinct. |
| D03 | 2026-07-10 | Use UTF-8 GitHub-flavored Markdown as the canonical representation; never store editable HTML or editor JSON as a second source of truth. | Accepted | Portable, searchable, exportable, and safer to render. |
| D04 | 2026-07-10 | Provide manual Save plus debounced autosave, IndexedDB write-ahead recovery, optimistic versions, idempotency receipts, and explicit conflict handling. | Accepted | “No content lost” cannot rely on overlapping last-write-wins network requests. |
| D05 | 2026-07-10 | Make notes searchable immediately through note FTS and asynchronously available to semantic retrieval, Ask, related items, and future graph refresh. | Accepted | Save durability must not depend on an external AI provider. |
| D06 | 2026-07-10 | Preserve provenance (`original`, `manual_note`, `ai_summary`) in results and citations; never present a personal interpretation as source fact. | Accepted | Required for user trust and accurate AI behavior. |
| D07 | 2026-07-10 | Keep the parent item as the graph node. Manual-note semantics may influence edge strength but do not create a separate note node in v1. | Accepted | Avoids visual/data duplication while still using personal context. |
| D08 | 2026-07-10 | Exact search is always available. AI/connection inclusion defaults on only when all note-consuming providers are verified local; any remote provider requires one-time provider-named acknowledgement first. A per-note opt-out is synchronously enforced in retrieval/provider eligibility before asynchronous artifact cleanup. | Accepted in v2 | Satisfies AI usefulness without sending a “private” note remotely before an informed action. |
| D09 | 2026-07-10 | Define “private” as HMAC-authenticated, same-owner, non-shared/default-export-excluded data—not end-to-end encrypted; disclose browser/server/local-backup/off-site-backup/provider boundaries. | Accepted in v2 | Matches current single-user architecture without a false encryption claim. |
| D10 | 2026-07-10 | Every current/future single, bulk, and share/export path defaults `include_notes=false`; explicit note export/inclusion requires authentication, preview/warning, and provenance. | Accepted in v2 | Prevents accidental disclosure while preserving deliberate portability. |
| D11 | 2026-07-10 | Ship a minimal recent-recovery surface; retain at most 25 server checkpoints and 30 days, plus per-editor dirty/conflict local drafts until acknowledged/discarded or declared local retention. | Accepted in v2 | Makes retained recovery data usable and does not pretend a losing local draft is a server revision. |
| D12 | 2026-07-10 | Merge attested consolidated production snapshot `8178117` and use migrations 021+ before implementation. | Accepted, release-blocking | Live distinguishing public-file hashes and migrations 018/019/020 match 8178117; stale main/4d97c45 are incomplete. |
