## Remaining cost levers (ranked)

**1. Switch to Hetzner CAX11 (~$4.10/month, EU) — saves $5.90/month, effort: low**
The only lever that materially cuts the bill. Accept ~280 ms additional Ask latency vs Mumbai. Implementation: spin up CAX11, copy credentials, update DNS — 2 hours work. Do this if the bill ever feels high, or before v0.7.0 if Mumbai latency proves to not matter. ROI: 2 hours of engineering saves $70.80/year.

**2. Raise enrichment skip threshold from <300 to <800 chars — saves ~$0.01/month, effort: trivial**
Short items (links with no body, quick notes) don't benefit from enrichment. At Brain's volume, savings are $0.01/month. Not worth doing now; worth remembering if volume hits 10×. One-line change in `enrichment-worker.ts`.

**3. Aggressive prompt trimming — drop `quotes` field from enrichment output — saves ~$0.005/month, effort: low**
`quotes` field adds ~40 output tokens per item. At 30 items/month: 1,200 tokens × $2.50/MTok = $0.003/month saved. Do not bother at this scale; `quotes` is useful UX.

**4. Everything else: don't bother**
Switching embedding models (saves $0), switching enrichment to Gemini Flash (saves $0.04/month), caching embeddings for dedup (saves < $0.01/month at 30 captures), switching Ask to Haiku (saves $0.18/month but degrades quality). All have negative ROI: the engineering hour to implement any of them costs $100 and saves under $2.40/year at current usage. The bill is $10.26. Squeeze the VM if you need to squeeze anything.

## Alerting setup

| Service | Alert type | Threshold | Action |
|---|---|---|---|
| **Anthropic** | Monthly hard cap | **$5.00/month** | API returns 429 when cap is hit; enrichment queues up, Ask fails gracefully. Set in console.anthropic.com → Usage → Spending Limits. |
| **Anthropic** | Soft warning | **$3.00/month** | Email notification at 60% of hard cap. Investigate: unexpected enrichment volume, runaway batch resubmit loop, or API key leak. |
| **AWS Lightsail** | CloudWatch billing alarm | **$15.00/month** | Console alert + email. Covers VM cost ($10) + unexpected data transfer or snapshot charges. Set in AWS Billing Console → Budgets. |
| **Backblaze B2** | Storage size | **500 MB bucket total** | Manual check monthly; no native alert on free tier. At 35 MB/month growth, 500 MB = ~14 months runway. Add B2 lifecycle rule to delete backups older than 30 days. |
| **Lightsail bandwidth** | Transfer usage | **1.0 TB used** (of 1.5 TB included) | Lightsail console shows transfer gauge. At < 1 GB/month normal usage, hitting 1 TB means something is wrong (runaway log dump, misconfigured client hammering the API). |

Policy: Anthropic hard cap at $5.00 is 3× the $1.63 AI-only expected monthly spend at 10× usage — it only fires on a genuine runaway. Lightsail alarm at $15.00 is 1.5× the $10 fixed cost — fires on any unexpected overage.

## Stop-squeezing threshold

**Below $12.00/month, further cost optimization is negative ROI.**

One engineering hour = $100 in opportunity cost. Any lever that saves less than $100/year (= $8.33/month) requires 12+ months to break even on a single hour of work. At the current $10.26 bill, only the Hetzner switch ($5.90/month saving = $70.80/year) breaks even in under 18 months. Every other lever on this list does not.

Concretely: if the monthly bill stays under $12 (i.e., within $1.74 of today), do nothing. Revisit only if Lightsail raises prices or usage scales 10×+ and the AI line crosses $3.00/month.

## DDoS / viral scenario sanity-check

Brain is single-user and unauthenticated requests are cheap by design.

`GET /api/health` — confirmed in S-7 runbook: returns HTTP 401 `{"error":"Unauthorized"}` immediately. No DB reads, no LLM calls, no vector lookup. Cost: zero beyond the TCP handshake through the Cloudflare tunnel.

All routes that do expensive work (`/api/ask`, `/api/items`, `/api/items/[id]/enrich`) check the bearer token in the first middleware step and return 401 before touching SQLite or calling any API. An unauthenticated flood of requests burns Cloudflare's edge capacity (which is built for this) and costs nothing on the backend.

**Verdict:** A 1,000-request public flood costs $0.00 in AI API spend and $0.00 in additional compute. The Cloudflare tunnel absorbs it. No DDoS cost risk.
